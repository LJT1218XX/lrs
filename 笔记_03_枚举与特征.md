# 笔记 3：枚举（enum）+ 匹配（match）+ 特征（trait）

> 对应 Rust Book ch6（enum + match）、ch10.2（trait）

## 一、枚举（enum）

### 对比 C

| C `enum` | Rust `enum` |
|----------|------------|
| `enum E { A, B }` 纯整数 | `enum E { A, B(i32) }` 每个变体可带数据 |
| `switch` 漏分支不报错 | `match` 强制穷举所有变体 |
| 类型不安全（可赋任意整数） | 类型安全 |

### 简单 enum

```rust
enum TrafficLight {
    Red,
    Yellow,
    Green,
}

let light = TrafficLight::Red;

let action = match light {
    TrafficLight::Red => "停",
    TrafficLight::Yellow => "等",
    TrafficLight::Green => "行",
};
```

### 带数据的 enum（Rust 的核心特性）

```rust
enum IpAddr {
    V4(u8, u8, u8, u8),  // 4 个 u8
    V6(String),           // 一个 String
}

let home = IpAddr::V4(127, 0, 0, 1);

let desc = match home {
    IpAddr::V4(a, b, c, d) => format!("{}.{}.{}.{}", a, b, c, d),
    IpAddr::V6(s) => format!("IPv6: {}", s),
};
```

**关键：** C 的枚举每个变体只是整数，Rust 的枚举每个变体可以是不同类型——这叫**代数数据类型**（Algebraic Data Type）。

### match 必须穷举

```rust
match val {
    VariantA => ...,
    VariantB => ...,
    // 少写一个变体 → 编译错误！
}
```

### if let — 只关心一种变体

```rust
// match 写法：
match msg {
    Some(content) => println!("{content}"),
    _ => {},
}

// if let 简化：
if let Some(content) = msg {
    println!("{content}");
}
```

### 解构（destructure）

`match` / `if let` 的变体名后面跟的变量就是在解构提取内部数据：

```rust
match ip {
    IpAddr::V4(a, b, c, d) => ...  // 解构出 4 个 u8
    IpAddr::V6(s) => ...            // 解构出 String
}
```

### 命名规范

- 枚举名：**大驼峰**（`PascalCase`） → `TrafficLight`、`IpAddr`
- 变体名：**大驼峰** → `Red`、`V4`

## 二、特征（trait）

### 类比

| 语言 | 概念 | 说明 |
|------|------|------|
| Rust | `trait` | 定义共享行为 |
| Java | `interface` | 类似，但 Rust trait 更灵活 |
| C++ | 纯虚类 | 类似，但无继承开销 |

### 定义与实现

```rust
trait Greet {
    fn greet(&self) -> String;          // 只声明，不实现
    fn say_hello(&self) -> String {     // 默认方法（带实现）
        String::from("你好！")
    }
}

struct Dog { name: String }
struct Cat { name: String }

impl Greet for Dog {
    fn greet(&self) -> String {
        format!("{}说：汪汪！", self.name)
    }
    // say_hello 用默认实现
}

impl Greet for Cat {
    fn greet(&self) -> String {
        format!("{}说：喵喵！", self.name)
    }
    fn say_hello(&self) -> String {     // 覆盖默认方法
        format!("{}说：你好（高冷）", self.name)
    }
}
```

### trait 作为函数参数

```rust
// 接受任何实现了 Greet 的类型
fn make_sound(animal: &impl Greet) {
    println!("{}", animal.greet());
}

// 等价写法（完整泛型语法）：
fn make_sound<T: Greet>(animal: &T) {
    println!("{}", animal.greet());
}
```

两种写法等价，后者在需要多次引用同一个泛型类型时使用。

### trait 的价值

```rust
// 没有 trait：每加一种动物写一个新函数
fn greet_dog(d: &Dog) { ... }
fn greet_cat(c: &Cat) { ... }

// 有 trait：一个函数处理所有
fn greet_animal(a: &impl Greet) { ... }
```

## 三、泛型与分发方式

### 泛型（Generics）

```rust
// T 是类型参数，调用时编译器替换为具体类型
let a: Option<i32> = Some(42);
let b: Option<String> = Some(String::from("hello"));
```

编译器为每种具体类型生成一份独立代码 → **单态化**（monomorphization）。

**零运行时开销：** 编译时已展开，和手写具体类型一样快。

### 静态分发 vs 动态分发

| | `&impl Trait` | `&dyn Trait` |
|--|---------------|--------------|
| 调度时机 | 编译期 | 运行时（vtable） |
| 性能 | 零开销，可内联 | 函数指针跳转 |
| 代码体积 | 每种类型一份 | 通用一份 |
| 适用 | 类型编译期已知 | 类型运行时才确定 |

```rust
// 静态分发：不能用不同类型的对象
// let shapes: Vec<&impl Area> = vec![&Circle{}, &Rect{}]; // ❌

// 动态分发：可以混用不同类型
let shapes: Vec<&dyn Area> = vec![&Circle{}, &Rect{}]; // ✅
```

## 四、lrs 项目重构

### 重构历程

1. **原始代码：** `if-else` 判断 `is_dir()` / `is_symlink()` → 两处重复（`type_char` 和 `display_name`）
2. **引入 enum：** `enum EntryKind` + `classify()` 统一分类
3. **引入 trait：** `trait FileDisplay` 提供 `icon_char()` 和 `colorize()` 方法

### 最终结构

```
EntryKind enum            → 分类标准
  ├── classify() 函数     → 把 FileType + 后缀 映射为 EntryKind
  └── FileDisplay trait   → 根据 EntryKind 返回图标字符 + 颜色
        └── list_dir()    → 调用 kind.icon_char() / kind.colorize()
```

### 加新文件类型只需改 3 处

1. `enum EntryKind` 加变体（如 `Archive`）
2. `classify()` 加 `else if`
3. `impl FileDisplay` 加颜色

`list_dir` 里不需要改——这就是 **trait 封装变化** 的典型应用。
